# Diagrams — Solution Design

## Purpose
Diagrams describe the functional flow of the solution.
They are used for discussion with the client and as reference for development.

## Rules
- Use Excalidraw format only (`.excalidraw`).
- No screenshots or exported images.
- Diagrams must remain editable.
- No technical implementation details.
- Separate happy path and error paths.

## Naming convention
- `<process_name>_overview.excalidraw`
- `<process_name>_error_paths.excalidraw`

## Level of detail
- Focus on steps, decisions, and outcomes.
- Do not reference classes, methods, or libraries.
