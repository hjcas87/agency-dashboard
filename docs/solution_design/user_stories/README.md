# User Stories — Solution Design

## Rules
- One user story per file.
- One user story maps to exactly one automation use case.
- No technical language.
- No UI selectors or tool names.

## Naming convention
- `US-<short_slug>.md`
  - example: `US-import-transactions.md`

## Required sections
- Title
- Description
- Preconditions
- Happy Path
- Alternate / Error Paths
- Acceptance Criteria
